# 标准草案：南京旅游景点数据采集规范（基于多源UGC与XOOTOUR架构）

> 本版本基于 XOOTOUR 景点系统架构，引入 L1~L5 数据度量层，面向南京六朝古都与博爱之都特色定制。

---

## 一、核心设计原则

```
1. 面向「外宾体验与南京国际历史文化名城形象」：提取国际通用的游玩决策，突出六朝文化底蕴与民国风情。
2. 面向「多源趋势引领」：利用马蜂窝的游记攻略与打卡机位、携程的结构化门票数据、TripAdvisor的外宾真实评分，反向筛选南京最具历史厚度与人文温度的景点。
3. 面向「全流程闭环」：严密结合结构化表单（基础、画廊、亮点、路线、设施、套餐），确保前台100%完美还原。
4. 所有字段必须可结构化 + 可 AI 理解：方便通过大模型自动从多源数据中抽提信息。
```

**数据治理原则：**

```
5. 多源校验：以马蜂窝作为主要 UGC 及图片源，高德地图作为坐标与基础地址源，携程/官方作为门票与须知核验源，TripAdvisor作为外宾视角补充源。
6. 数据血缘：每条景点记录需标注对应的数据来源链接或官方源，可追溯采集时间。
7. 合规优先：图片需进行去水印、去版权争议处理，内容通过 AI 进行二次转写拒绝全文抄袭。
```

---

## 二、数据分层结构（结合 XOOTOUR 系统板块）

```
L1：基础与概况层（必须）     —— 主表 tour_spot + 概况表 tour_spot_info
L2：游玩与须知层（核心）     —— 开放时间、交通方式、参观须知、门票套餐
L3：出片与打卡层（差异化）   —— 马蜂窝游记热度快照、推荐打卡季节、核心受众标签
L4：AI 与深度内容层（体验）  —— 亮点体验 + 游玩路线 + 服务设施
L5：质量与血源层（可信度）   —— 多源数据追踪、置信度评估、更新周期控制
```

### 核心表/子表联动设计

| 后台菜单版块 | 系统对应数据表名 | 对应采集/关联说明 |
|-------------|----------------|------------------|
| 景点列表 | `tour_spot` (主表) | 核心基础数据：名称、副标题(英文)、SEO描述、基础价格、主图、所属城市ID（南京） |
| 景点列表 (已融合) | `tour_spot_info` (概况表) | 详情介绍、详细地址、开放时间、交通方式、参观须知 |
| 景点画廊 | `tour_spot_banner` (子表) | 轮播图，利用马蜂窝/官方高清图，每个景点最少5张 |
| 亮点体验 | `tour_spot_feature` (子表) | 由 AI 从马蜂窝游记热点词中提炼亮点体验 |
| 游玩路线 | `tour_spot_route` (子表) | 推荐游览路线，整合马蜂窝"保姆级路线" |
| 设施服务 | `tour_spot_facility` (子表) | 停车场、游客中心、WIFI等 |
| 景点套餐 | `tour_spot_package` (子表) | 门票价格及套餐信息 |
| 分类配置 | `tour_spot_cate` | 分类标签：民国记忆、六朝文脉、陵寝古迹、秦淮风情、博物馆等 |

## 三、标准字段定义

### L1：基础与概况层

| 字段名 | 类型 | 必须 | 说明 / 示例 |
|--------|------|------|------------|
| `spot_id` | INT | ✅ | 主键 ID |
| `name_zh` | VARCHAR(200) | ✅ | 中文全名（如：中山陵、夫子庙-秦淮河、南京博物院、总统府） |
| `name_en` | VARCHAR(200) | ✅ | 英文全名（如：Dr. Sun Yat-sen's Mausoleum、Confucius Temple） |
| `city_id` | INT | ✅ | 固定关联南京的城市ID |
| `district` | VARCHAR(100) | ✅ | 行政区（玄武区、秦淮区、鼓楼区、建邺区、栖霞区等） |
| `address_zh` | TEXT | ✅ | 中文详细地址 |
| `address_en` | TEXT | ✅ | 英文详细地址 |
| `lat` | DECIMAL(10,7) | ✅ | 纬度（WGS84） |
| `lng` | DECIMAL(10,7) | ✅ | 经度（WGS84） |
| `cover_img` | VARCHAR(500) | ✅ | 封面图 URL |
| `seo_desc` | VARCHAR(500) | ✅ | SEO 描述 |
| `summary_zh` | TEXT | ✅ | 详情介绍（中） |
| `summary_en` | TEXT | ✅ | 详情介绍（英） |

### L2：游玩与须知层

| 字段名 | 类型 | 必须 | 说明 / 示例 |
|--------|------|------|------------|
| `open_hours` | TEXT | ✅ | 开放时间（如中山陵"08:30-17:00 周一闭馆"、夫子庙"全天开放"） |
| `transportation` | TEXT | ✅ | 交通方式：地铁/公交（如"地铁2号线苜蓿园站"） |
| `visit_notice` | TEXT | ✅ | 参观须知：中山陵预约制、侵华日军南京大屠杀遇难同胞纪念馆着装要求等 |
| `base_price` | DECIMAL(10,2) | ✅ | 起步参考价格（0.00 代表免费开放） |
| `rating` | DECIMAL(2,1) | ✅ | 综合评分 |

### L3：出片与打卡层

| 字段名 | 类型 | 必须 | 说明 / 示例 |
|--------|------|------|------------|
| `ugc_notes_count` | INT | ❌ | 马蜂窝相关游记热度指标 |
| `best_visit_season` | VARCHAR(100) | ❌ | 最佳游玩季节（如："春秋/气候宜人"、"秋季/栖霞山红叶"、"冬季/秦淮灯会"） |
| `photo_spots` | JSON | ❌ | 最佳打卡机位 |
| `crowd_tags` | JSON | ❌ | 适合人群标签：`["Solo", "Couple", "Family-friendly", "History Buff", "Photography"]` |

### L4：AI 与深度内容层

| 子表 | 关键字段 | 说明 |
|------|----------|------|
| `tour_spot_banner` | image_url, sort_order, photo_credit | 景点画廊多图（最少5张） |
| `tour_spot_feature` | title, description | 亮点体验（如："中山陵392级台阶"、"秦淮灯影夜泊"） |
| `tour_spot_route` | route_name, duration_hours, route_nodes, description | 游玩线路（如："钟山风景区一日线"） |
| `tour_spot_facility` | facility_name, is_bold, description | 服务设施 |

### L5：质量与血源层

| 字段名 | 类型 | 必须 | 说明 |
|--------|------|------|------|
| `source_platform` | VARCHAR(50) | ✅ | 数据源平台标识 |
| `source_note_ids` | JSON | ❌ | 数据源链接/ID列表 |
| `data_collected_at` | TIMESTAMP | ✅ | 数据抓取时间 |
| `data_quality_score` | DECIMAL(3,2) | ❌ | 数据完整度评分 0-1 |
| `is_ai_generated` | BOOLEAN | ✅ | 是否经由 AI 结构化清洗转写 |

## 四、南京景点筛选标准

📥 **收录标准（必须满足 ≥2 条）**

1. 马蜂窝包含"南京 [景点名]"关键词的高赞游记（点赞 > 100）数量不少于 8 篇。
2. 属于南京核心名片（中山陵、夫子庙秦淮河、南京博物院、总统府、明孝陵、玄武湖等）。
3. 具备独特的六朝古都历史底蕴或民国文化辨识度。
4. TripAdvisor 上有对应的英文景点页面及外宾评分。

❌ **排除标准**

- 在马蜂窝近期大量被标记为"照骗/踩雷/已关闭"的场所。

## 五、数据源优先级

| 优先级 | 数据平台 | 核心获取内容 |
|--------|---------|------------|
| **P0** | 马蜂窝 (Mafengwo) | UGC 游记、景点攻略、打卡机位、高清图片 |
| **P0** | 高德地图 API | 标准名称、WGS84坐标、标准行政区划与地址 |
| **P1** | 携程 (Ctrip) | 门票套餐、开放时间、参观须知、用户点评 |
| **P2** | TripAdvisor | 英文景点介绍、外宾点评与评分 |

## 六、AI 与脚本处理流程

### Step 1: 自动化采集
马蜂窝抓取 Top 20 热门游记 → 携程抓取门票结构化数据 → TripAdvisor抓取英文点评 → 高德API获取地理数据。

### Step 2: AI 结构化提取
大模型解构为六大板块数据（详情介绍、画廊图片、亮点体验、游玩路线、设施服务、门票套餐）。

### Step 3: 后台导入与绑定
API创建景点主数据（绑南京城市ID）→ 循环写入了表完成全自动关联绑定。
