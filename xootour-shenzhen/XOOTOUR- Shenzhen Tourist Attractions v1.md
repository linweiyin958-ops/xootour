# 标准草案：深圳旅游景点数据采集规范（基于多源UGC与XOOTOUR架构）

> 本版本基于 XOOTOUR 景点系统架构，引入 L1~L5 数据度量层，面向深圳创新活力之城与滨海休闲特色定制。

---

## 一、核心设计原则

```
1. 面向「外宾体验与深圳国际化创新都市形象」：提取国际通用的游玩决策，突出科技创新、主题乐园与滨海城市特色。
2. 面向「多源趋势引领」：利用马蜂窝的游记攻略与打卡机位、携程的结构化门票数据、TripAdvisor的外宾真实评分，反向筛选深圳最具活力和美感的景点。
3. 面向「全流程闭环」：严密结合结构化表单（基础、画廊、亮点、路线、设施、套餐），确保前台100%完美还原。
4. 所有字段必须可结构化 + 可 AI 理解：方便通过大模型自动从多源数据中抽提信息。
```

**数据治理原则：**

```
5. 多源校验：以马蜂窝作为主要 UGC 及图片源，高德地图作为坐标与基础地址源，携程/官方作为门票与须知核验源，TripAdvisor作为外宾视角补充源。
6. 数据血缘：每条景点记录需标注对应的数据来源链接或官方源，可追溯采集时间。
7. 合规优先：图片需进行去水印、去版权争议处理，内容通过 AI 进行二次转写拒绝全文抄袭。
```

---

## 二、数据分层结构（结合 XOOTOUR 系统板块）

根据 XOOTOUR 规范与数据分层逻辑，景点数据被拆解为以下五层：

```
L1：基础与概况层（必须）     —— 主表 tour_spot + 概况表 tour_spot_info（标识、深圳行政区、地理坐标）
L2：游玩与须知层（核心）     —— 开放时间、交通方式、参观须知、门票套餐（tour_spot_package）
L3：出片与打卡层（差异化）   —— 马蜂窝游记热度快照、推荐打卡季节、核心受众标签
L4：AI 与深度内容层（体验）  —— 亮点体验（tour_spot_feature）+ 游玩路线（tour_spot_route）+ 服务设施（tour_spot_facility）
L5：质量与血源层（可信度）   —— 多源数据追踪、置信度评估、更新周期控制
```

### 核心表/子表联动设计

| 后台菜单版块 | 系统对应数据表名 | 对应采集/关联说明 |
|-------------|----------------|------------------|
| 景点列表 | `tour_spot` (主表) | 核心基础数据：名称、副标题(英文)、SEO描述、基础价格、主图、所属城市ID（深圳） |
| 景点列表 (已融合) | `tour_spot_info` (概况表) | 详情介绍、详细地址、开放时间、交通方式、参观须知 |
| 景点画廊 | `tour_spot_banner` (子表) | 顶部的多张轮播图，利用马蜂窝/官方高清图，每个景点最少5张 |
| 亮点体验 | `tour_spot_feature` (子表) | 景点特色、亮点体验描述，由 AI 从马蜂窝游记热点词中提炼 |
| 游玩路线 | `tour_spot_route` (子表) | 推荐游览路线及关键打卡点信息，整合马蜂窝"保姆级路线" |
| 设施服务 | `tour_spot_facility` (子表) | 停车场、游客中心、WIFI、无障碍设施等，标注加粗项 |
| 景点套餐 | `tour_spot_package` (子表) | 门票价格及打包游玩套餐信息（部分免费景点可为空） |
| 分类配置 | `tour_spot_cate` | 定义深圳景点的分类标签（如：主题乐园、滨海休闲、科技创新、自然风光、购物商圈等） |

## 四、深圳景点筛选标准

📥 **收录标准（必须满足 ≥2 条）**

1. 马蜂窝包含"深圳 [景点名]"关键词的高赞游记（点赞 > 100）数量不少于 5 篇。
2. 属于深圳核心城市名片（世界之窗、欢乐谷、深圳湾公园、大梅沙、东部华侨城等）或华侨城主题公园群。
3. 具备独特的滨海休闲特色（较场尾、杨梅坑、西涌海滩等）或科技文化创新标签（华强北、南山科技园周边创意园区等）。
4. TripAdvisor 上有对应的英文景点页面及外宾评分。

❌ **排除标准**

- 在马蜂窝/TripAdvisor上近期大量被标记为"照骗/踩雷/已关闭/正在大修"的场所。
- 纯工业园区或不对公众开放的科技企业园区。
- 已关闭拆除或长期维修无明确恢复开放时间的景点。

## 五、数据源优先级

在深圳区域，数据采集与校验的优先级配置如下：

| 优先级 | 数据平台 | 核心获取内容 |
|--------|---------|------------|
| **P0** | 马蜂窝 (Mafengwo) | 主要 UGC 源：深度游记、景点攻略、推荐路线、打卡机位、真实体验避坑、高清图片。 |
| **P0** | 高德地图 API | 基建核心源：标准中文名称、高精度 WGS84 经纬度坐标、标准行政区划与标准地址。 |
| **P1** | 携程 (Ctrip) | 业务核验源：官方标准门票套餐、结构化开放时间、参观须知、用户点评。 |
| **P2** | TripAdvisor | 外宾补充源：英文景点介绍、外国游客真实点评与评分、外宾关注点。 |

## 六、AI 与脚本处理流程

### Step 1: 自动化采集

- **马蜂窝**：针对目标深圳景点，抓取景点详情页及 Top 20 热门游记，获取图片、攻略路线、打卡机位及真实避坑指南。
- **携程**：抓取景点门票页结构化数据（开放时间、门票套餐、参观须知、用户评分）。
- **TripAdvisor**：抓取英文景点页面，获取外宾评分、英文介绍及真实英文点评。
- **高德地图**：通过 API 获取标准坐标、地址、行政区等结构化数据。

### Step 2: AI 结构化提取

大模型接收多源数据流，直接解构为系统所需的六大板块数据（详情介绍、画廊图片、亮点体验、游玩路线、设施服务、门票套餐）。

### Step 3: 后台导入与绑定

1. 脚本通过 API 先行在 `tour_spot` 和 `tour_spot_info` 中创建景点主数据，绑定深圳城市 ID。
2. 脚本自动循环调用附属表 API，在各子表中填入主键 ID，完成全自动的数据强关联绑定。
